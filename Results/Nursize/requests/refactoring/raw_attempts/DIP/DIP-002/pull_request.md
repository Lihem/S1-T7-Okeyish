- **Severity**: high
- **Confidence**: detected by 12 scan(s)

**Description.** Session directly instantiates concrete HTTPAdapter objects for transport.

**Reasoning.** The `Session` class, a high-level orchestrator for HTTP requests, directly creates instances of `HTTPAdapter` and mounts them for 'https://' and 'http://' schemes. This hardcodes the transport mechanism, making the `Session` tightly coupled to the concrete `HTTPAdapter` implementation. If a different transport adapter (e.g., for custom protocols, mocking, or alternative HTTP libraries) were needed, the `Session` class itself would require modification, violating DIP's principle that high-level modules should depend on abstractions (e.g., an `ITransportAdapter` interface) rather than concrete implementations.

## Fix logic

- **Model**: `gemini-3.1-flash-lite-preview` (temperature 0.2)
- **Patch blocks**: 1
- **Files modified** (1): `src/requests/sessions.py`

**Summary.** Inject default HTTPAdapter instances into Session via constructor to improve testability and decouple dependencies.

**Rationale.** The Session class was hardcoding the instantiation of HTTPAdapter. By allowing these to be passed as optional arguments (defaulting to the original behavior), we enable dependency injection, allowing users or tests to provide mock or custom adapters without modifying the Session class.
