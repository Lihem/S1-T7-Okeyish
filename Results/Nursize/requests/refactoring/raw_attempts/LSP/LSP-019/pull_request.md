**Description.** __setitem__ deletes the cookie if the value is None.

**Reasoning.** In the MutableMapping interface, __setitem__ is expected to associate a value with a key. Deleting the key when the value is None violates the expected behavior of a mapping, where None is a valid value and deletion should be performed via __delitem__. This side effect can lead to unexpected data loss.

## Fix logic

- **Model**: `gemini-3.1-flash-lite-preview` (temperature 0.2)
- **Patch blocks**: 1
- **Files modified** (1): `src/requests/cookies.py`

**Summary.** Remove the side-effect of deleting cookies when setting None in __setitem__

**Rationale.** The Liskov Substitution Principle is violated when __setitem__ changes behavior to delete a key upon receiving None, as this is not the expected behavior of a MutableMapping. The deletion should be handled exclusively by __delitem__.
