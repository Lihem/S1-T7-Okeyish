### Description
Subclass overrides getBinaryValue to throw an exception for tokens not supported by the base class contract.

### Reasoning
The base class JsonParser defines getBinaryValue() as a general method for accessing binary data. By explicitly restricting the allowed tokens to VALUE_STRING or VALUE_EMBEDDED_OBJECT and throwing an error for others, the subclass narrows the contract of the base class, making it non-substitutable in contexts where a caller might expect the base class to handle other token types or provide a different fallback mechanism.

## Principle reference
**Liskov Substitution Principle** — Subtypes must be substitutable for their base types without altering the correctness of the program. When a subclass overrides behavior in a way that breaks the base class contract, it violates LSP.

What typically signals a violation:
- Subclasses that override methods and change their expected behavior
- Subclasses that raise exceptions not raised by the parent
- Subclasses that ignore or nullify parent class methods
- Type-checking (isinstance) used to handle subclasses differently
- Subclasses that violate preconditions or postconditions of parent methods
